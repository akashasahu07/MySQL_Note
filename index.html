MySQL Commands

MySQL commands are divided mainly as 4 sub languages.

  • DDL(Data Definition language): These commands help us to structure our database
    • Example: CREATE, ALTER, RENAME, DROP, TRUNCATE etc.
  • DML(Data Manipulation Language): These commands help us to manipulate or update or delete our data.
    • Example: INSERT, UPDATE, DELETE etc.
    • These commands also contain sub part called
    • DQL(Data Query Language): It's command is select which is used for queries.
  • DCL(Data Control Language): These commands helps to control the data which user manipulate.
    • Example: GRANT, REVOKE etc.
  • TCL(Transaction Control Language): These commands helps to control the transactions and undo the previous query.
    • Example: COMMIT, ROLLBACK, SAVEPOINT etc.

DDL(Data Definition Language)

These commands are used to build the structure of database.

  • create: Create is used to create databases, tables, views etc.
    • Database: Syntax: CREATE DATABASE "database-name";
    • Table: Syntax: CREATE TABLE "table-name(column1 datatype,
      column2 datatype,
      column3 datatype, ...)";
      • Example:CREATE TABLE students( sid int, sname varchar(20), sfee decimal(10,2) );
  • Rename: Rename is used to rename the table, view etc.
    • Syntax: RENAME TABLE "old-table-name" TO "new-table-name"
    Example: RENAME TABLE teachers TO faculty;
  • Alter: Alter is used to add/modify/delete columns from database table
    • Add A Column:
      Syntax: ALTER TABLE "table-name" ADD COLUMN "column-name" datatype;
      Example: -- Add column gender into the table
      ALTER TABLE students ADD COLUMN gender char(1);

      If you need to add a column in a specific location like after a perticular column we want to add a column then

      Example:
      -- After sname add phone Number
      ALTER TABLE students ADD COLUMN phno int after sname;
    • Rename Column:
      Syntax: ALTER TABLE "table-name" CHANGE "old-col-name" "new-col-name" datatype;
      Example: ALTER TABLE faculty CHANGE tsalary salary decimal(10,2);
    • Delete a Column:
      Syntax: ALTER TABLE "table-name" DROP COLUMN "column-name";
      Example: ALTER TABLE faculty DROP COLUMN salary;
  • Drop: Drop is used to delete a table/column/database etc.
    • Syntax: -- Delete a database/table
    DROP DATABASE "database-name"/"table-name"
    Example: DROP DATABASE school;
    DROP TABLE students;
  • Truncate: Truncate is used to delete only data from the table but table will be there..
    • Syntax: TRUNCATE TABLE "table-name";

Q: What is the difference between drop, truncate, and delete command ?

They are used to remove data, but they function differently.

  • DELETE: Removes specific rows from a table based on a WHERE condition. If no WHERE clause is specified, all rows are deleted.
    • Syntax: DELETE FROM "table-name" WHERE condition;
  • TRUNCATE: Removes all rows from a table.
    • Syntax: TRUNCATE TABLE "table-name";
  • DROP: Completely removes the table from the database, including its structure.
    • Syntax: DROP TABLE "table-name";

Constraints

In MySQL, tables consists of columns which need to follow certain conditions. The conditions or rules set to the columns are called Constraints.

  • UNIQUE: Any column which contains UNIQUE constraints must have unique values for every record. Unique also accepts NULL values empty ones are filled with NULL value.
  • NOT NULL: Any column which contains NOT NULL constraint must contain values which are not null. Required fields.
  • AUTO INCREMENT: Any column which contains auto increment will increment the value of column automatically.
  • DEFAULT: Any column which contains default constraint can provides a value default to all rows. If user forget to add value it will add the default value to that row.
  • CHECK: Any column which contains CHECK constraints is used to check all values in that column are satisfying a condition or not. If i set user should enter their age should greater than 18 then only age that greater than 18 should accepted.
  • PRIMARY KEY: Primary key must be unique and not null value. It is a column by which we can identify each and every row individually.
  • FOREIGN KEY: Foreign Key is a Primary key in another table and got as a reference to this table.

create table employees (

age int check(age >= 18), country varchar(20) default "India",

Joins

  • Joins are used to combine two or more number of tables and retrive the data based on common field connection.
  • If you want to join two table the first table you can create like this. It must have foreign key and references.
    • Example: create table Orders(order_id int primary key, item varchar(20) not null, amount int not null, customer_id int not null, foreign key(customer_id) references Customers(customer_id) );
  • There are different joins existed in MySQL.
    • Inner Join: Inner Join is used to retrive the data which is common on both the tables. Both table common data will come.
      • Syntax: select t1.field1, t2.field2,... from table1 as t1 inner join table2 as t2 on t1.common-field = t2.common-field;

      Q: WAQ to retrive first_name, item they bought?

      Ans: select t1.first_name, t2.item from Customers as t1 inner join orders as t2 on t1.customer_id = t2.customer_id

      Q: WAQ to print customers name with total amount they spent?

      Ans: select t1.first_name, sum(t2.amount) from customers as t1 inner join orders as t2 on t1.customer_id = t2.customer_id group by t1.first_name;
    • Left Join: Left Join is used to retrive the data which is available in laft table and common data. All the people from left will come and only common from right table come.
      • Syntax: select t1.field1, t2.field2,... from table1 as t1 left join table2 as t2 on t1.common-field = t2.common-field;
    • Right Join: Right Join is used to retrive the data which is available in right table and common data. All the people from right will come and only common from left table come.

      • Q: WAQ to print every customer name with products they bought?

      Ans: select t1.first_name, t2.item from customers as t1 left join orders as t2 on t1.customer_id = t2.customer_id;
    • Full Outer Join: In MySQL, full outer join is not possible but can be achieved by 'UNION'.
      • Syntax: left join query UNION right join query;

'Having' Clause

  • Having is used in combination with the group by clause. It will work after the group by has worked and where is worked before group by.

    • Q: WAQ to print the list of countries who have more than 3 employees?

    Ans: select country, count(country) from employees group by country having count(country) > 3;

    Q: WAQ to divide students based on gender avg their marks print only their average is greater than 85?

    Ans: select gender, avg(smark) from students group by gender having avg(smark) > 85;

Difference between 'Where' and 'Having'

  • 'Where' will be applied before group by. 'Having' will be applied after group by.
  • 'Where' cann't handle aggregate functions. 'Having' can handle aggregate function.
    • Example: count(), sum() etc.
  • 'Where' and 'Having' both can use and /or and used for condition only.

Indexes

  • Indexes are used to speed up data retrival operations on database tables. It helps to retrive the data faster.

  • By default index will be created for primary key.

  • Delete Index:

Normalization

  • It is the process of organizing data in tables/databases to minimize data redundancy and improve data integrity.
  • 1NF: Each cell in a table must contain atomic value(only one value).
  • 2NF: Each non-key attribute must depend on primary key.
  • 3NF: If a table is in 2NF and in case non-key attribute depend on other values than primary key. Split into smaller tables.

Introduction

  • MongoDB is a "NoSQL" document based schema less database that allows you to store and manage large amount of semi-structured data.
  • MongoDB is a open-source NoSQL, document based schema less, BSON database which is used to storage, manage, scale large more efficiently.
  • It is flexible, scalable and easy to use.
  • There are so many NoSQL databases like Cassandra, couchDB, apacheDB etc..

Q: What are the differences between MongoDB and MySQL?

Ans:

  • MongoDb is document based structure and MySQL is table based structure.
  • MongoDB is schemaless database and MySQL contains the schema.
  • MongoDB is a NoSQL but MySQL uses SQL for database.

    • updateOne: It is used to update the document based on a condition. This method can update only one value.
      • Syntax: db."collection-name".updateOne(conditionObject, updationObject);
      Ex: db.students.updateOne( { Name: "Peter" }, { $set: { age: 26 } } );
    • updateMany: In case multiple elements need to updated. We can use updateMany. It is used to update multiple documents at a time.
      • Syntax: db."collection-name".updateMany(conditionObject, updationObject);
      Ex: db.students.updateMany( { Name: "Peter" }, { $set: { age: 26 } } );
  • Delete: Delete command is used to delete a specific conditioned document/documents. We have two ways to do that.
    • deleteOne: It is used to delete only one document which matches specific criteria.
      • Syntax: db."collection-name".deleteOne("conditionObject");
      Ex: db.actionMovies.deleteOne({title: "Avangers Endgame"});
    • deleteMany: It is used to delete all the documents which matched specific criteria.
      • Syntax: db."collection-name".deleteMany("conditionObject");
      Ex: db.actionMovies.deleteMany({year: 2019});
  • Find: Find is used to read/display the documents that matched specific criteria, we can do it in two ways.
    • findOne: It is used to read only one document that matched given criteria.
      • Syntax: db."collection-name".findOne("filteredObject");
    • find: It is used to read all the documents that matched given criteria.
      • Syntax: db."collection-name".find("filteredObject");

Operators

  • operators are used to filter the data using some conditions.
  • In MongoDB, operators are preceeded by "$".

  • Comparison Operators: These operators are used to compare values in queries.
    • $eq: Matches values that a equal to a specific value.
      • Ex: db.actionMovies.find({title: {$eq: "Titanic"}});
    • $gt: Matches values that are greater than a specific value.
      • Ex: db.actionMovies.find({year: {$gt: 2005}});
    • $lt: Matched values that are less than a specific value.
      • Ex: db.actionMovies.find({year: {$lt: 2005}});
    • Similarly we can also perform gte, lte, ne.
    • $in: Matches values that exists in an array of specified values.
      • Ex: db.actionMovies.find({title: {$in: ["Titanic", "Interstellar"]}});
    • $nin: Matches values that are not exists in an array of specified values.
      • Ex: db.actionMovies.find({title: {$nin: ["Titanic", "Interstellar"]}});
  • Logical Operator: These operators are used to combine multiple conditions.
    • $and: Matches documents that satisfy all the conditions.

      • Q: WQA to find whose age is greater than 20 and less than 30?
      Ans: db.students.find({$and: [{age: {$gt: 20}}, {age: {$lt: 30}}]});

    • $or: Matches documents that satisfy any one of the condition.
      • Ex: db.students.find({$or: [{age: {$gt: 20}}, {age: {$lt: 30}}]});
    • $not: Inverts the effects of query expression.
      • Ex: db.students.find({age: { $not: { $gt: 20, $lt: 30 } }});
    • $nor: Matches the documents that are failed in all conditions.
      • Ex: db.students.find({$nor: [{age: {$gt: 20}}, {age: {$lt: 30}}]});
  • Element Operator: Used to check weather a field is exist or not.
    • Syntax: db."collection-name".find({"field-name": {$exists: true}});
    Ex: db.students.find({age: {$exists: true}});
  • Assignment Operator: These operations are used to update the fields based on a condition.
    • $set: Set is used to update a field with a specified value.
      • Ex: db.students.updateOne({Name: "Vaani"}, {$set: {msg: "I Love You"}});
    • $unset: It is used to remove a certain field from a document.
      • Ex: db.students.updateOne({Name: "Vaani"}, {$unset: {msg: "I Love You"}});
    • $inc: Increment a numerical value.
      • Ex: db.students.updateOne({Name: "Vaani"}, {$inc: {age: 1}});
      db.students.updateOne({Name: "Vaani"}, {$inc: {age: -1}});
    • $mul: Multiplies a numerical value.
      • Ex: db.students.updateOne({Name: "Vaani"}, {$mul: {age: 2}});
    • $rename: Renames a field.
      • Ex: db.students.updateOne({Name: "Vaani"}, {$rename: {Name: "Baby"}});
  • We can use some default functions such as:
    • sort(): It is used to sort the results of a query in ascending or descending order. "1" is for ascending and "-1" is for descending.
      • Ex: db.students.find().sort({age: 1}); #Ascending order
      db.students.find().sort({age: -1}); #Descending order
    • Find with Projection: We can set second argument for find to select only which fields to display in query result.
      • Ex: db.students.find({}, {age: 1, _id: 0}).sort({age: 1}); #Only age will come nothing else
    • count(): it is used to count the total documents result in the query.
      • Syntax: "query".count();
      Ex: db.students.find({}, {age: 1, _id: 0}).sort({age: 1}).count();
    • limit(): it is used to limit the result to certain documents.
      • Syntax: "query".limit(value);
      Ex: db.students.find({}, {age: 1, _id: 0}).sort({age: 1}).limit(3);
    • skip(): it is used to skip the certain documents.
      • Syntax: "query".skip(value);
      Ex: db.students.find({}, {age: 1, _id: 0}).sort({age: 1}).limit(3).skip(1); #The first one is skipped
